Creating Databases in Compute Engine

1 hour 15 minutes Free

Overview

In this lab, you create database servers running on Compute Engine. You create both Linux and Windows servers and use the CLI to automate the creation of a server.

Objectives

In this lab, you learn how to perform the following tasks:

  • Create a MySQL database on Linux.

  • Create a SQL Server database on Windows.

  • Automate server creation using the Google Cloud SDK.

Task 1. Create a MySQL database on Linux

  1. Open a new web browser window and navigate to the Google Cloud Console (console.cloud.google.com). Use the project selector to choose the first project with a leading name of 'qwiklabs-gcp.'

  2. On the Navigation menu (), click Compute Engine > VM instances.

  3. Click Create instance.

  4. For Name, type mysql-db.

  5. Under Boot disk, click Change.

  6. For Operating system, select Debian, for Version, select Debian GNU/Linux 11 (bullseye) x86/64, and then click Select.

  7. When the VM is ready, click SSH.
    This will log you in to the server in a new tab.

  1. In the terminal window, to update the packages and install MySQL using the apt-get package manager, enter the following command:

sudo apt-get update sudo apt-get install -y default-mysql-server

  1. To secure the database, enter the following command:

sudo mysql_secure_installation

  1. When you are prompted for the current password for root, press Enter (Return) for none.

  2. When you are prompted for Switch to unix_socket authentication [Y/n], press N.

  3. When you are prompted for Change the root password? [Y/n], press Y.

Enter something you will remember, because you will need it later.

  1. When you are prompted for Remove anonymous users? [Y/n], press N.

  2. When you are prompted for Disallow root login remotely? [Y/n], press N.

  3. When you are prompted for Remove test database and access to it? [Y/n], press N.

  4. When you are prompted for Reload privilege tables now? [Y/n], press Y.

  5. Now log in to the database. Enter the following command, and then enter your password when prompted:

sudo mysql -u root -p

  1. At the mysql> prompt, to see the current databases, enter the following command:

SHOW databases;

  1. To create a new database, enter the following command:

CREATE database petsdb;

  1. Switch to that database:

USE petsdb;

  1. Create a table:

CREATE TABLE pets ( id INT AUTO_INCREMENT PRIMARY KEY, name VARCHAR(255), breed VARCHAR(255) );

  1. Add a record:

INSERT INTO pets (name, breed) VALUES ('Noir', 'Schnoodle');

  1. Confirm that your record was added:

SELECT * FROM pets;

  1. Type exit to log out of the database.

  2. Type exit again to close the SSH session.

Task 2. Create a SQL Server database on Windows

  1. Return to the Google Cloud Console and the Compute Engine service.

  2. Click Create instance.

  3. For Name, type sql-server-db.

  4. For Machine type, select e2-standard-4.

  5. Under Boot disk, click Change.

  6. For Operating system, select SQL Server on Windows Server, for Version, select SQL Server 2019 Web on Windows Server 2019 Datacenter, and then click Select.

  7. Accept the remaining defaults, and then click Create.

  8. When the Windows server is ready, click the dropdown arrow in the Connect column, and then select Set Windows password.

Note: If you get to this point too quickly, the Windows server may not be ready, even though a green check in the console indicates that it is. Wait a couple of minutes and try again.

  1. Leave the Username as it is (the generated username is a portion of your Qwiklabs username), and then click Set.

  2. Copy and paste the generated password into a text file so you don't lose it, and then click Close.

You need to have an RDP client to log on to the Windows machine. 11. Do one of the following:

  • If you are using Windows, you already have RDP:

    • Click the dropdown arrow in the Connect column, and then select Download the RDP file.
    • When the download is complete, double-click the file and then log in with your username and the password you copied.

  • If you are using a Mac:

    • Install the Microsoft Remote Desktop client from the Apple App Store (you may already have it installed).
    • After you install the client, download the RDP file, and then log in with your username and the password you copied.

  1. When you are logged on to the Windows server, close the Server Manager Dashboard.

  2. On the Start menu, type SSMS, then right click the SQL Server Management Studio shortcut, and choose Run as administrator to run the program.

  3. On the Popup window choose Yes when prompted Do you want to allow this app to make changes to your device?.

  4. In the Connect to Server dialog, the server name should be SQL-SERVER-DB. Click Connect.

  5. In Management Studio, in the Object Explorer, right-click SQL-SERVER-DB, and select Properties.

  6. In the Server Properties dialog, select the Security page.

  7. For Server Authentication, select SQL Server and Windows Authentication mode, and then click OK.

  8. Read the warning message and click OK again.

  9. In Object Explorer, right-click the server, and select Restart.

  10. Click Yes when prompted. If the Restart command is disabled, click the Start menu and reboot the server. Give it a minute or two and then restart your RDP session.

  11. When you're logged in, return to SQL Server Management Studio and connect to the database server.

  12. Expand the Security and Logins branch of the tree view. Right-click on the Logins branch, and select New Login.

  13. For Login name, enter your first name or another unique value.

  14. Select SQL Server authentication.

  15. Enter a password you will remember (paste it in a text file if you like).

  16. Clear Enforce password policy.

    a. Click the Server roles page, and select the sysadmin role.

    b. Click the User Mapping page, and select all the databases.

    c. Click OK.

  17. To test the login you just created, in Object Explorer, click Connect, and then select Database Engine.

  18. For Authentication, select SQL Server Authentication.

  19. Enter your username and password, and then click Connect.

  20. In the toolbar, click New Query.

  21. To create a database, enter the following command, and then click Execute:

CREATE DATABASE petsdb;

  1. In the databases dropdown to the left of the Execute button, select your petsdb database.

  2. To create a table, enter the following command, and then click Execute:

USE petsdb; CREATE TABLE pets ( id INT PRIMARY KEY IDENTITY (1, 1), name VARCHAR (MAX), breed VARCHAR (MAX) );

  1. To add a record:

INSERT INTO pets (name, breed) VALUES ('Noir', 'Schnoodle');

  1. To confirm that it worked:

SELECT * FROM pets;

  1. Close your RDP connection.

Review

You just created a SQL Server database server running on Windows.

Task 3. Automate server creation using the Google Cloud SDK

  1. Return to the Google Cloud Console and the Compute Engine service.

  2. Click Create Instance.

  3. For Name, type db-server.

  4. Under Boot disk, click Change.

  5. For Version, select Debian GNU/Linux 11 (bullseye) x86/64, and then click Select.

  6. Expand Advanced options then Management. In Automation - Startup script paste the following:

#! /bin/bash apt-get update apt-get install -y default-mysql-server
  1. Scroll to the bottom, but do not click the Create button; instead click the Equivalent Code button and then navigate to the Command Line pane on the right of the screen.

This pane shows you the gcloud CLI command you can use to automate this server's creation as you configured it. Next, make a small change to this command.

  1. Click Copy at the bottom of the pane.

  2. Cancel the Create an instance wizard.

  3. In Notepad or similar text editor, paste the copied code.

  4. If present, from the list of commands remove the following parameter: --provisioning-model=STANDARD

Note: Please be careful to only delete the provisioning-model parameter.

  1. Select the entire edited command and copy it to the clipboard.

  2. Launch the Cloud Shell in the top right section of the Cloud Console page, paste the command, and press Return/Enter to create the VM.

If prompted, click Authorize.

  1. Press Return (Enter) to allow for an empty passphrase.

  2. Please wait for 3 to 5 minutes for the startup script to run. Then in the Cloud Shell terminal, enter the following to log in to the virtual machine:

gcloud compute ssh db-server --zone=us-central1-a

  1. If asked: Do you want to continue(Y/n)?, Enter Y. Accept all defaults for setting up secure communications.

  2. To see whether your database server is running, enter the following command:

sudo systemctl status mysql

The output on the console should contain a row similar to the following:

Aug 17 14:32:10 db-server systemd[1]: Started MariaDB 10.5.15 database server.
  1. Press Q to exit the status screen.
